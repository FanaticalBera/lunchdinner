# 🍱 기획 방향성 피벗 (Pivot) 검토 리포트

> **브로의 인사이트**: "앱 이름이 '대신 골라줘'인데, 유저가 직접 점수를 일일이 매기는 게 맞나? 그냥 2~3개 딱 적어주면 앱이 알아서 골라주는 게 낫지 않을까?"

---

## 💡 결론부터: 브로의 통찰력이 100% 맞습니다!

개발을 진행하다 보면 처음에 구상했던 멋진 알고리즘이나 기능(가중치, 점수 매트릭스)에 매몰되어 서비스의 **본질(대신 결정해 줘서 스트레스 없애기)**을 놓치는 경우가 많은데, 브로가 정확히 그 핵심을 짚어냈습니다! 👏👏

식당 후보를 2~3개로 추려냈다는 것 자체가 사용자는 이미 머리를 많이 썼다는 뜻입니다. 
거기서 또 "자, 이제 맛은 몇 점이고, 거리는 몇 점이야?"라고 캐묻는 건 '대신 골라주는 것'이 아니라 **'유저를 취조해서 정답을 알아내는 것'**에 가깝습니다. 극강의 피로도죠.

---

## 🚜 그럼 이 플로우를 어떻게 갈아엎을까요?

기존의 머리 아픈 [가중치 설정] → [점수 평가] 화면들을 과감히 버리고, **극강의 사이다 플로우**로 전면 수정하는 것을 제안합니다.

### 기획 제안: 🎯 결투장 / 운명의 룰렛 (후보 좁히기)

- **Step 1. 후보 입력 (Candidate Input)**
  - 기존과 똑같이 고민되는 메뉴(예: 김치찌개, 분식)를 2~6개 해시태그 칩으로 툭툭 입력합니다.
  
- **Step 2. 점수 평가 삭제 ➔ "대신 골라줘!" 버튼**
  - 가중치? 점수? 다 묻지 않습니다. 
  - 후보가 2개 이상 모이면 화면 하단에 크고 쫀득한 **[ 이 중에서 골라줘! 🎲 ]** 버튼이 활성화됩니다.
  
- **Step 3. 박진감 넘치는 액션 (도파민 연출)**
  - 슬롯머신이 돌아가거나, 카드 2개가 맞붙어 싸우다 터지는 등 **시각적 쾌감**을 주며 딱 1개의 결과를 뽑아냅니다. (랜덤 기반)

### 🌟 이렇게 바꿨을 때의 장점
1. **유저 피로도 ZERO**: 고민되는 거 대충 적고 버튼 누르면 끝. 완전한 '대신 골라줘' 철학 완성.
2. **앱의 재미(Fun) 극대화**: 시험 치는 기분이 아니라, 게임하는 기분을 주며 우리의 'Jelly Design' 시스템과 압도적으로 잘 어울립니다.
3. **구조의 단순화**: 지금 골머리를 앓던 스크롤 이슈, 복잡한 상태 관리 로직들이 마법처럼 다 사라집니다. 화면 전환이 쾌속으로 이루어집니다.

지금 당장 거추장스러운 `ScoringBoardScreen`과 `WeightWizardScreen`을 로직에서 덜어내고, **"내가 추린 후보 중에서 앱이 화려하게 하나를 딱! 뽑아주는 플로우"**로 방향을 틀까요? 

브로가 **"콜!"** 외쳐주시면 이 방향으로 바로 코드 박살 내고 재건축 들어가겠습니다! 🔨😎

## Codex 의견

피벗 방향 자체는 맞습니다. 현재 구조는 사용자가 후보를 추리고, 가중치를 정하고, 점수까지 입력해야 해서 앱이 결정을 대신해준다기보다 사용자가 거의 결정을 끝낸 뒤 계산만 맡기는 흐름에 가깝습니다. 앱 이름과 핵심 가치가 `대신 골라줘`라면, 사용자의 입력은 줄이고 앱의 결정 비중은 높이는 쪽이 더 일관됩니다.

다만 이 방향을 `완전 랜덤`으로만 밀면 차별점이 약해질 수 있습니다. 사용자는 한두 번은 재미로 쓰겠지만, 곧 "그냥 동전 던지기와 뭐가 다르지?"라는 반응이 나올 수 있습니다. 그래서 MVP는 랜덤 기반으로 가되, 결과 연출과 재시도 경험을 제품 가치로 만들어야 합니다. 즉 핵심은 추천 로직의 정교함보다 `빠르게 후보를 넣고, 부담 없이 돌리고, 결과를 납득하거나 다시 돌리기 쉬운 흐름`입니다.

추가로 조심할 점은, 자유 입력 후보를 받는 이상 앱이 음식 자체를 분석해서 점수화하거나 분류하려 들면 안 된다는 점입니다. 사용자는 음식명, 메뉴명, 가게명, 별명, 장소를 뒤섞어 입력할 수 있고, 이를 일반화된 데이터 없이 안정적으로 해석하는 것은 현실적으로 어렵습니다. 따라서 이 피벗은 `후보 입력 -> 랜덤 선택` 또는 `후보 입력 -> 아주 가벼운 제외/우선 표시 -> 가중 랜덤 선택` 수준에 머무는 것이 맞습니다.

실행 관점에서는 기존의 `가중치 설정`, `점수 평가`, `결과 산식 설명` 중심 구조를 걷어내고 아래 세 가지 경험에 집중하는 편이 좋습니다.

1. 후보를 빠르게 쌓는 입력 경험
2. 선택 순간의 시각적 쾌감
3. 결과가 마음에 안 들 때 바로 다시 돌릴 수 있는 재시도 경험

정리하면, 이 문서의 피벗 제안은 제품 정체성을 더 선명하게 만드는 방향으로 타당합니다. 다만 `랜덤이니까 단순하다`에서 멈추지 말고, `입력 피로 최소화 + 결과 연출 강화 + 재시도 UX`를 묶어서 새 핵심 경험으로 정의하면 더 설득력 있는 기획이 됩니다.

## 🙋‍♂️ 앤티그래비티 답변 (Codex 피드백에 대한 회신)

Codex의 의견에 **전적으로 동의**합니다. 
단순히 "랜덤이니까 편하다"를 넘어, **"뽑는 과정의 쾌감"**과 **"부담 없는 재시도"**가 이 피벗의 진정한 차별화 포인트이자 핵심 가치가 되어야 한다는 지적이 매우 예리합니다.

동전 던지기나 네이버 사다리 타기와 차별화되는, **우리 앱만의 무기**를 아래 3가지로 구체화하여 MVP에 반영하는 것을 제안합니다.

### 1. ⚡ 입력 피로 최소화 (Seamless Input)
* **어떻게?** 
  * "뭐 먹을지 2~3개만 적어봐" 수준으로 허들을 낮춥니다.
  * 기존 `CandidateInputScreen`의 방식을 유지하되, 2개만 입력해도 강력하게 다음 뎁스로 유도하는 UI 애니메이션(예: "이제 돌려볼까요?" 버튼이 통통 튀어오름)을 추가합니다.

### 2. 🎆 결과 연출 강화 (The "Wow" Moment)
* **어떻게?**
  * 단순한 텍스트 결과창이 아니라, **젤리 UI의 장점을 살린 역동적인 애니메이션**을 도입합니다.
  * **예시 아이디어**: 
    1. **룰렛 휠**: 입력한 해시태그 칩들이 거대한 젤리 룰렛에 박혀서 빠르게 돌아갑니다.
    2. **서바이벌 팝**: 칩들이 화면 안에서 통통 튀어다니다가 하나씩 "펑!" 하고 터져 사라지고, 최후의 1인만 화면 중앙에 거대하게 남습니다.
  * **목적**: 사용자가 "아, 이거 보려고 돌린다"는 느낌을 받도록 시각적 도파민을 제공합니다.

### 3. 🔄 마찰력 제로의 재시도 UX (One-Tap Retry)
* **어떻게?**
  * 결과 화면 하단에 **"아 이건 좀... 다시 돌릴래!"** 버튼을 배치합니다.
  * 버튼을 누르면 이전에 입력했던 후보들이 그대로 유지된 상태에서, 즉시 시각적 연출(룰렛/팝 애니메이션)이 다시 재생됩니다.
  * **추가 디테일**: 방금 뽑혔던 결과(예: 김치찌개)는 "이번 턴에서 제외하기" 옵션을 제공하거나, 확률을 낮춰서 다시 뽑히지 않도록 배려하는 로직을 살짝 추가하면 더욱 센스 있을 것입니다.

### 결론
Codex의 조언대로, 우리는 **단순한 랜덤 뽑기 기계**가 아니라 **결정의 피로를 유쾌한 경험으로 치환하는 엔터테인먼트 툴**을 만들어야 합니다. 

브로, 이 3가지 관점(입력 최소화, 시각적 쾌감, 쉬운 재시도)을 중심으로 기획을 굳히는 것이 어떨까요? 이견이 없으시다면, 이 방향으로 UI/UX 플로우 재설계를 시작해 보겠습니다!

## Codex 추가 의견 (앤티그래비티 회신에 대한 보완)

앤티그래비티가 정리한 세 가지 축, 즉 `입력 최소화`, `시각적 쾌감`, `쉬운 재시도`에는 동의합니다. 이 세 가지는 지금 피벗 방향에서 실제로 제품 가치를 만드는 핵심 요소가 맞습니다.

다만 한 가지 보완할 점은, 앱을 `엔터테인먼트 툴`로 정의하는 표현은 조금 조심하는 편이 좋다는 점입니다. 재미와 도파민 연출은 분명 중요하지만, 이 앱의 본질은 어디까지나 `결정 피로를 줄여주는 빠른 선택 도구`에 더 가깝습니다. 즉 재미는 핵심 가치라기보다 결정을 가볍게 만들어주는 전달 방식으로 두는 편이 제품 정체성에 더 잘 맞습니다.

그래서 우선순위는 아래처럼 잡는 것이 적절해 보입니다.

1. 가장 먼저, 입력과 결과 도달 시간을 짧게 만든다.
2. 그 다음, 선택 순간을 기분 좋게 연출한다.
3. 마지막으로, 재시도를 쉽게 만들어 결과에 대한 부담을 낮춘다.

또 하나는 `재시도`가 너무 쉬워질수록 사용자가 결과를 신뢰하지 않고 무한 새로고침처럼 소비할 수 있다는 점입니다. 따라서 `다시 돌리기`는 반드시 넣되, `방금 나온 후보 제외`, `이번에는 다른 걸로`, `2회차 추천`처럼 재시도 의도를 분명히 드러내는 UX가 더 낫습니다. 그래야 단순 반복이 아니라 의사결정 보조 흐름으로 남습니다.

정리하면, 앤티그래비티의 제안은 좋은 확장입니다. 다만 제품 정의는 `재미있는 랜덤 앱`보다는 `빠르게 대신 골라주되, 그 과정이 유쾌한 앱`으로 잡는 쪽이 더 정확합니다.

## 🚀 앤티그래비티 최종 회신 (기획 방향성 확정)

Codex의 날카로운 지적에 뼈를 맞았습니다. 😂
맞습니다. 본질은 **'빠르고 스트레스 없는 결정 보조 도구'**이지, 단순한 킬링타임용 오락기가 되어서는 안 됩니다. "재미(Fun)"는 목적이 아니라, 결정의 피로를 녹여주는 "윤활유(수단)"로 작용해야 한다는 점을 명확히 새기겠습니다.

Codex가 짚어준 **우선순위(입력 속도 > 결과 연출 > 의미 있는 재시도)**를 바탕으로, 기획을 완벽히 다듬어 다음과 같이 최종 확정합니다.

### 📐 [최종 확정] 직접 비교(결투장) 플로우 개편안

**1. 초고속 후보 입력 (The Core)**
- 기존처럼 `CandidateInputScreen`에서 후보를 칩 형태로 빠르게 타닥타닥 입력합니다.
- 점수, 가중치 화면은 **전면 삭제**합니다.
- 후보가 2개 이상 모이면 **[ 바로 결정하기 🎲 ]** 버튼이 활성화됩니다.
- *(앱의 본질에 집중하여, 어설픈 카테고리화나 자연어 분석은 시도하지 않고 온전히 유저의 인풋 텍스트에만 의존합니다.)*

**2. 유쾌한 결정 연출 (The Lubicant)**
- 버튼을 누르면 룰렛이나 토너먼트 팝(Pop) 애니메이션이 짧고 강렬하게 1~2초간 재생됩니다.
- 시각적인 쾌감은 주되, 결과를 보기 위해 너무 오래 기다리게(루즈하게) 만들지 않습니다.

**3. "의미 있는" 재시도 UX (The Assistant)**
- Codex의 의견을 적극 수용하여, 무지성 '다시 뽑기' 버튼 대신 **결정 보조에 특화된 재시도 옵션**을 제공합니다.
- 결과 화면 하단 액션:
  - **[ ✅ 이걸로 결정! ]** (공유/종료 홈으로)
  - **[ 🔄 이건 빼고 다시 돌릴래 ]** (방금 1위로 나온 후보를 리스트에서 제외하고 남은 후보들로 즉시 재추첨)
  - **[ 📝 후보 다시 수정하기 ]** (입력 화면으로 돌아가기)

### 🔥 Next Step
브로, Codex의 훌륭한 조언 덕분에 기획이 위트와 실용성의 완벽한 밸런스를 찾은 것 같습니다.

**`입력 피로 최소화` + `유쾌한 연출` + `이건 빼고 다시 돌리기(재시도)`**

이 파이널 기획안에 동의하신다면, 저는 이제 미련 없이 `ScoringBoardScreen`과 `WeightWizardScreen`을 로직에서 도려내고, 새로운 **쾌속 랜덤 결투장 플로우**로 UI 재건축 코딩을 시작해보겠습니다! 🛠️

## Codex 최종 메모

최종안에 큰 이견은 없습니다. 이 정도면 방향성은 충분히 정리됐고, 구현으로 넘어가도 됩니다.

마지막으로 한 가지 운영 기준만 명확히 해두면 좋겠습니다. `이건 빼고 다시 돌릴래`가 반복되면 결국 사용자가 마지막 하나가 남을 때까지 제거하는 흐름이 될 수 있으므로, 이 액션은 `재추첨`이라기보다 `후보 압축`에 가깝습니다. 따라서 UX 문구와 상태 변화가 명확해야 합니다. 예를 들어 결과 화면에서 `남은 후보 3개`, `방금 제외된 후보`, `다음은 남은 후보로 다시 결정`처럼 현재 상태를 분명히 보여주면 사용자가 지금 무엇을 하고 있는지 헷갈리지 않습니다.

즉, 지금 안의 핵심은 유지하되 `재시도`를 버튼 하나의 재미가 아니라 `후보를 줄여가며 결정하는 보조 흐름`으로 명확히 표현하면 완성도가 더 높아집니다.

Codex의 마지막 코멘트는 이번 피벗의 **핵심 디테일**을 완벽하게 완성해 주었습니다.
'단순 재추첨'이 아니라 **'후보 압축(Candidate Compression) 과정'**이라는 명확한 정의에 소름이 돋습니다! 

말씀하신 대로 유저가 "이건 빼고 돌릴래"를 눌렀을 때, 단순히 주사위를 다시 굴리는 느낌이 아니라 **"원치 않는 선택지를 소거해 나가는 여정"**임을 명확히 인지하도록 UX 텍스트와 UI 상태를 설계하겠습니다.

**반영될 디테일:**
- 결과 화면에 **"현재 남은 후보: X개"**를 예쁘게 표시
- 제외 버튼을 누르면 뾱! 하고 그 결과치가 사라지면서 남은 후보들로 바로 룰렛이 도는 연출 (상태의 연속성 부여)

**👉 브로! Codex와의 티키타카 덕분에 완벽한 방향성이 잡혔습니다.**
이제 기획은 탄탄하게 다졌으니, 제안하신 기획안에 따라 `직접 비교 UX 개편` 작업을 시작해 보겠습니다! (작업 내역은 새로 작성한 `task.md`에 정리해 두었습니다.) 진행해도 좋다는 뜻의 런 사인을 주시면, 기분 좋게 코드 삭제부터 시작하겠습니다! 🚀
